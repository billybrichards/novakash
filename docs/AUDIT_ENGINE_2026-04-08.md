# Engine Architecture Audit — April 8, 2026

## Critical Issues

### C1: GTC fills after 60s polling window create orphaned positions
- After 60s fill-check poll, trade marked EXPIRED in DB
- But CLOB order alive until GTD expiry (window_ts + duration + 120s)
- If filled between 60s and GTD expiry: position exists on Polymarket, no DB tracking
- **Fix**: Don't mark EXPIRED until GTD expiry confirmed, or extend poll window

### C2: Position monitor fuzzy-matches trades by cost (±$1)
- Three separate DB queries with different matching criteria
- Can link wrong trade if multiple trades have similar stakes
- **Fix**: Match by token_id or condition_id from metadata

### C3: trade_bible table not populated by engine
- Created manually via SQL, not in any engine code
- Documented as "source of truth" but never written to
- **Fix**: Either auto-populate from trades or remove documentation references
