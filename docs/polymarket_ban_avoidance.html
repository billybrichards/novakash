The 12 Rules for Bot Survival

Rule 1: NEVER Trade from a Blocked Country
  • ✗ Don't deploy to Railway Europe (routes through DE/NL/AT)
  • ✗ Don't use a UK residential IP
  • ✓ Deploy to a VPS in Japan, South Korea, Brazil, India, or UAE
  • ✓ Check GET https://polymarket.com/api/geoblock from your server BEFORE enabling live trading
  • ✓ Add a startup geoblock check to the engine — refuse to start in live mode if blocked
Rule 2: ONE Asset, ONE Order at a Time
  • ✗ Don't fire 5 orders across 3 assets in 2 seconds
  • ✗ Don't trade multiple timeframes (5m + 15m) simultaneously
  • ✓ Trade ONE asset per 5-minute window
  • ✓ Add 30-60 second delays between orders on different assets
  • ✓ Prioritise: pick the best signal, trade only that one
Rule 3: Humanise Your Order Pattern
  • ✗ Don't stake $3.431912785 (7 decimal precision = bot)
  • ✗ Don't always use the exact same entry timing (T-60.000s)
  • ✓ Round stakes to $0.50 increments ($3.50, $4.00, $5.00)
  • ✓ Add random jitter: T-55s to T-65s instead of exact T-60s
  • ✓ Vary order sizes ±15% randomly
Rule 4: Rate Limit EVERYTHING
  • ✗ Don't retry immediately on failure
  • ✗ Don't poll order status every 5 seconds
  • ✓ Maximum 1 order per 30 seconds
  • ✓ Maximum 10 orders per hour
  • ✓ Add exponential backoff on any 4xx error
  • ✓ If order fails, wait 5+ minutes before retrying
Rule 5: No +2¢ Retry Pattern
  • ✗ Don't cancel and re-submit at a higher price — classic bot signal
  • ✗ Don't auto-bump prices in a tight loop
  • ✓ Place one order at a fair price. If it doesn't fill, accept it.
  • ✓ If you must retry, wait 60+ seconds and treat it as a new order
Rule 6: Warm Up New Wallets
  • ✗ Don't go from zero activity → 50 automated trades
  • ✓ First week: 1-3 manual trades per day via the web UI
  • ✓ Second week: 5-10 trades per day, mix of manual and API
  • ✓ Third week+: gradually increase API volume
  • ✓ Always maintain some manual web UI activity on the account
Rule 7: Use Residential/Mobile Proxies
  • ✗ Don't trade from datacenter IPs (AWS, Railway, GCP, etc.)
  • ✓ Route API calls through a residential proxy in an allowed country
  • ✓ Rotate proxy IPs periodically (but not too fast — that's suspicious too)
  • ✓ Services: Bright Data, Oxylabs, SmartProxy (residential rotating)
Rule 8: Separate Read and Write Endpoints
  • ✓ Read operations (Gamma API, order book, prices) → direct from server, high frequency OK
  • ✓ Write operations (place order, cancel) → through residential proxy, rate limited
  • ✓ This reduces proxy costs and suspicious write-pattern exposure
Rule 9: Use GTD (Good Till Date), Not GTC
  • ✓ Set orders to expire in 60-120 seconds (GTD with short window_ts)
  • ✓ Reduces the "stale bot order" pattern of GTC orders sitting on the book
  • ✓ Already implemented in v5 — keep this
Rule 10: Don't Trade Every Window
  • ✗ Don't trade 288 times per day (every 5-min window)
  • ✓ Maximum 20-30 trades per day
  • ✓ Skip windows where confidence is only MODERATE — trade HIGH only
  • ✓ Add a per-hour trade limit to the engine (e.g., max 4/hour)
Rule 11: Multiple Wallets / Key Rotation
  • ✓ Maintain 2-3 funded wallets
  • ✓ Rotate between them (Wallet A for odd hours, Wallet B for even)
  • ✓ Never blast orders from the same wallet on multiple markets simultaneously
  • ✓ Fund from different source addresses (not all from the same exchange)
Rule 12: Monitor and Circuit-Break
  • ✓ If ANY order returns 4xx, immediately halt all trading for 15 minutes
  • ✓ If 3 consecutive errors, halt for 1 hour and alert on Telegram
  • ✓ Track Polymarket's /api/geoblock status every 10 minutes
  • ✓ If wallet balance drops unexpectedly, halt and investigate
