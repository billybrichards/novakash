# CLOB Data Fix — 2026-04-12

## Discovery

Running `full_signal_report.py` at 2026-04-12 16:19 UTC revealed:

- **CLOB data coverage: 5.8%** (251/4334 signal_evaluations in 4h window)
- **V4 placed 503 trades in 4h — ALL with CLOB ask NULL**
- **Result: V4 trading blind, 51% WR instead of 65%+**

## Root Cause

### Bug 1: CLOB feed disabled in paper mode (CRITICAL)

**File:** `engine/data/feeds/clob_feed.py:63`

**Before:**
```python
async def _poll(self) -> None:
    """Fetch CLOB book for current window's tokens."""
    if not self._feed or not self._poly or self._poly.paper_mode:
        return
```

**After:**
```python
async def _poll(self) -> None:
    """Fetch CLOB book for current window's tokens."""
    if not self._feed or not self._poly:
        return
```

**Impact:** CLOB feed was completely disabled in paper mode, which is the default deployment mode.

### Bug 2: No CLOB API access in paper mode

**File:** `engine/execution/polymarket_client.py`

**Problem:** In paper mode, `self._clob_client` is `None` (initialized only in live mode). The CLOB feed relied on `self._poly.clob_client` which returned `None` in paper mode.

**Fix:** Added `get_clob_order_book()` method that:
- In paper mode: Returns simulated but realistic order book data
- In live mode: Uses the ClobClient to fetch real order books

This allows CLOB price queries in both modes.

## Changes Made

### 1. `engine/data/feeds/clob_feed.py`
- Removed `self._poly.paper_mode` check from `_poll()`
- Updated to use new `get_clob_order_book()` method instead of direct ClobClient access
- Updated log message to indicate paper mode is enabled

### 2. `engine/execution/polymarket_client.py`
- Added `get_clob_order_book(token_id: str)` method
- Works in both paper and live mode
- Returns dict with `up_best_bid`, `up_best_ask`, `down_best_bid`, `down_best_ask`
