# Clean Architecture Audit: Hub (FastAPI Backend)

**Date:** 2026-04-14  
**Scope:** `/Users/billyrichards/Code/novakash/hub/`  

## Executive Summary

The Hub maintains a reasonable **3-layer separation** (routes → services → DB), with **good use of dependency injection** for session management. However, three critical violations emerge: (1) **ad-hoc SQL migrations in main.py lifespan** bypass Alembic entirely, (2) **dashboard.py has 14+ bare `try/except: pass` blocks** swallowing errors silently, and (3) **routes directly hydrate DB models in responses**, leaking ORM implementation details to the API layer.

## Top 3 Priority Fixes

### 1. Remove Migrations from main.py Lifespan (CRITICAL)

**Problem:** DDL (CREATE TABLE, ALTER TABLE) hardcoded in `main.py:50-168` runs on every startup, bypassing Alembic versioning. No audit trail, no rollback capability.

**Solution:**
```python
# db/migrations/v_hub_v10_trading_config.py
async def upgrade(session):
    await session.execute(text("""
        CREATE TABLE IF NOT EXISTS trading_configs (
            id SERIAL PRIMARY KEY,
            ...
        )
    """))
    await session.execute(text("ALTER TABLE trades ADD COLUMN IF NOT EXISTS mode VARCHAR(16)"))
    # etc.

async def downgrade(session):
    await session.execute(text("DROP TABLE trading_configs"))

# main.py lifespan:
@asynccontextmanager
async def lifespan(app):
    log.info("hub.starting")
    await init_db()
    # Remove all raw SQL, rely on:
    # docker exec hub alembic upgrade head
    yield
    await close_db()
```

**Impact:** Enables schema versioning, rollback, deployment safety.

---

### 2. Log Exceptions Instead of Silent Failures (HIGH)

**Problem:** Bare `except: pass` or `except: return []` in dashboard/analysis routes hide bugs. If DB query fails, frontend gets empty data instead of an alert.

**Solution:**
```python
# Before (dashboard.py:64-86)
try:
    result = await session.execute(...)
    ...
except Exception:
    return []

# After
import structlog
log = structlog.get_logger(__name__)

try:
    result = await session.execute(...)
    ...
except Exception as e:
    log.error("dashboard.vpin_history_failed", error=str(e), exc_info=True)
    raise HTTPException(
        status_code=500,
        detail={"error": "vpin_history_failed", "error_id": str(uuid.uuid4())}
    )
```

**Impact:** Operational visibility; frontend can detect and display errors.

---

### 3. Decouple ORM from API Responses (HIGH)

**Problem:** Routes return raw Trade/Signal/DailyPnL ORM objects. If schema changes, API contract breaks. Consumers see SQLAlchemy internals.

**Solution:**
```python
# api/schema.py (new)
from pydantic import BaseModel

class TradeDTO(BaseModel):
    id: int
    order_id: str
    strategy: str
    venue: str
    direction: str
    stake_usd: float | None
    pnl_usd: float | None
    outcome: str | None
    created_at: str

    @staticmethod
    def from_orm(trade: Trade) -> "TradeDTO":
        return TradeDTO(
            id=trade.id,
            order_id=trade.order_id,
            ...
            created_at=trade.created_at.isoformat() if trade.created_at else None
        )

# api/trades.py
@router.get("/trades")
async def list_trades(...) -> dict:
    ...
    return {
        "trades": [TradeDTO.from_orm(t).model_dump() for t in trades],
        "total": total,
        ...
    }
```

**Impact:** API contract is stable; schema evolution becomes a service concern, not a breaking change.
